# MOEX Multimodal EOD Trading System

Мультимодальная система генерации торговых сигналов на дневном таймфрейме (EOD).

Система объединяет:
- Рыночные данные (OHLCV)
- Новости (заголовки → эмбеддинги)

Цель — предсказать вероятность роста цены на следующий торговый день и проверить, улучшают ли новости качество стратегии по сравнению с market-only моделью.

---

## Как работает система

После закрытия торгов система:

1. Загружает рыночные данные  
2. Загружает и обрабатывает новости  
3. Строит признаки  
4. Предсказывает p_up — вероятность роста  
5. Генерирует торговый сигнал  
6. Запускает walk-forward бэктест  
7. Сохраняет модель и метрики  

---

## Формулировка задачи

Целевая доходность:

r_{t+1} = log(C_{t+1} / C_t)

Бинарная классификация:

y_t = 1, если r_{t+1} > 0

Модель возвращает:

p_up ∈ [0, 1]

---

## Признаки

### Рыночные
- Доходности: r1, r5, r20  
- Волатильность: vol5, vol20  
- Дневной диапазон: (H − L) / C  
- Объём: log(1 + V)  

Используется временное окно 30–60 дней.

### Новостные (MVP)
- Заголовки новостей  
- Текстовый эмбеддинг  
- Усреднение по торговому дню  
- news_count  

---

## Торговая логика

LONG  если p_up > upper  
SHORT если p_up < lower  
CASH  иначе  

PnL учитывает комиссию и смену позиции (turnover).

---

## Оценка качества

Используется walk-forward тестирование (только time-based split).

Метрики:
- Sharpe Ratio  
- Max Drawdown  
- CAGR  
- Net PnL  
- Turnover  
- Стресс-тест (комиссия ×2)

Без утечки данных.  
Нормализация обучается только на train.
